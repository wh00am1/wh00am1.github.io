簡介

在程式執行開始之前，shell會做execve()以及fork()來啟動程式

接著程式就會被映射到記憶體上，然後開始接下來的一串動作:

  1. 找PT_INTERP segment
  2. map text, data, 其中text在lower address
  3. 載入ld.so, 映射到記憶體(process memory)
  4. 分配Bss
  5. 跳轉到OEP, 大多數程式應該都是_start

然後那隻被execve()的程式這時候才真的開始做事XD

Conclusion

看完之後大概整理程式啟動流程如下:

  1. 找PT_INTERP segment
  2. map text, data, 其中text在lower address
  3. 載入ld.so, 映射到記憶體(process memory)
  4. 分配Bss
  5. 跳轉到OEP, 大多數程式應該都是_start
  6. 進去_start之後她幫你呼叫__libc_start_main
  7. __libc_start_main呼叫init array完成初始化然後呼叫__libc_csu_init
  8. 終於執行main(argc, argv, envp)然後保存狀態exit()

Function Prologue and Epilogue

ASM Function prologue and Epilogue

Function Prologue

基本上是定義一些函數在初始化時需要做的事，就像在C裡面的int main(int argc, char *argv[])一樣是成文規定

Implementation

在Function Prologue中，主要做三件事:

  1. 在stack上保存現在這個stack frame的base poiner(例如RBP)
  2. 將base pointer對準stack address pointer(例如RSP)
  3. 分配stack空間

Function Epilogue

也是一個寫法上的規定，是函數結束時需要做的事(ex.回收stack之類的)，配合Prologue, 可以讓程式執行時為各函數分配stack空間，也不會互相損毀各自的stack frame

Implemetation

基本上是將Prologue的流程反過來執行一次:

  1. 將stack address pointer對準base pointer
  2. 恢復舊的stack base
  3. ret

而在x86架構下，有個更精簡的寫法:

就是利用leave 代替了mov rsp, rbp; pop rbp;

補充

既然寫了prologue, epilogue，那就順便寫寫call/ret好了

call

Intro

是用來呼叫函數使用的instruction, 主要做兩件事:

  1. 保存現在的Instruction pointer, 等下才可以return
  2. 將Instruction pointer設成目標函數 (相當於jmp)

Implementation

注意其實沒有push rip這個寫法，只是為了表示用，控制流程只能用call/ ret/ jxx

ret

Intro

在函數執行完畢返回時所執行的Instruction, 只會做一件事:

將保存在現在stack頂部(因為已經在epilogue末尾，base pointer已經被取走了)的return address 設成Instruction pointer

Implementation

同樣也是為了表示才用pop rip這種寫法 (這是不合法的)

補充知識

然後再補一下assembly jxx <address> instruction:

基本上是2 byte的instruction:

  1. 第一個byte是jxx指令本身
  2. 第二個byte是offset(相對於當前的RIP/EIP)

以0x00400767 的jne 0x400773為例:

  1. 此處機械碼為 750a
  2. 0x75 為 jne
  3. 0x400773 = 0x400767 + 0xa +2 (注意+2因為RIP已經讀完這兩個byte)

所以我們可以先把0x00400767 的jne 0x400773 改成跳到main裡面

這樣就可以成為無限迴圈(因為v4永遠不會等於255, 0xff 是無效的opcode)
