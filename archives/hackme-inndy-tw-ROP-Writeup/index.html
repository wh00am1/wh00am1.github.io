hackme.inndy.tw ROP & ROP2 (pwn)

ROP

保護機制

NX, Partial RELRO

分析

32-bit, 靜態編譯

程式邏輯

這題直接看組語便可解:

main

接著追進overflow() :

overflow

可以看到由於使用gets(), 造成明顯的overflow

利用方式

由於存在overflow且為靜態編譯, 可以嘗試組ROP chain:

ROPgadget --binary ./rop --ropchain

exploit

ROP2

保護機制

NX, Partial RELRO

分析

一樣32-bit, 但是動態編譯, ROPgadget可能不能使用

程式邏輯

首先看看組語:

main

跟上一題是差不多的邏輯，但多了syscall()

也可以在overflow()找到漏洞

利用方式

由於有syscall, 我們可以構造任意system call, 而且因為是x86, 所以不用找pop gadget

在binary中找不到/bin/sh, 所以可以先構造syscall把/bin/sh讀到記憶體中, 再想辦法getshell

想法:

  1. 構造read()把/bin/sh寫到Bss段
  2. 構造execve()執行execve(&Bss, 0, 0)

exploit

查一下syscall table可以找到:

  1. read = 3
  2. execve = 11

構造paylaod:

  1. syscall(3, 0, Bss, 8) 然後再跳轉回main
  2. syscall(11, Bss, 0, 0)

exploit:
